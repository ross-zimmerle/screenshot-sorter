# Contributing to Screenshot Sorter

We love your input! We want to make contributing to Screenshot Sorter as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style.
6. Issue your pull request!

## Code Style

- Use `gofmt` to format your code
- Write descriptive commit messages
- Comment your code where necessary
- Write meaningful tests

## Testing

Run the test suite with:
```bash
make test
```

For test coverage report:
```bash
make coverage
```

## Pull Request Process

1. Update the README.md with details of changes if needed.
2. Update the version number in relevant files.
3. Your PR will be merged once you have the sign-off of other developers.
